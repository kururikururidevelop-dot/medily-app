# 画面コンポーネント定義書

# コンポーネント設計方針

## 一貫性

* 医療系らしい **優しく清潔な UI** を統一する
* 白＋緑＋薄グレーを基本色として、情報構造の認識コストを下げる

## 再利用性
* すべての UI 要素は **Auto Layout** 前提で作成
* サイズ可変（レスポンシブ）のコンポーネントを基本とする

## アクセシビリティ
* 最小タップ領域：**44px**
* コントラスト比：4.5:1以上
* 色だけで意味を持たせない

# 外部サービス連携コンポーネント実装ガイド

## LINE/X との連携について

以下のコンポーネントは、Webアプリの画面定義では LINE/X 外部アプリ内での実装を想定しています：

### OptionButtonGroup（選択肢回答）
* **Webアプリ上での表示**：該当なし（Webでは表示専用）
* **LINE/X での実装**：
  - LINE Flex Message のボタンテンプレート
  - X では会話形式（ボタン選択肢）
  - タップで選択肢を確定し、システムへ送信

### 回答・一言・逆質問の送信フロー（LINE/X 内）
* 本コンポーネント定義書は **Webアプリ画面** を対象としています
* LINE/X 内での回答操作、一言送信、逆質問送信は、各プラットフォームの API（Messaging API、DM等）を使用して実装されます
* システムは LINE/X からの送信結果を受信し、Webアプリ上の「MessageCardList」「MessageCard」で表示します

### ExternalButton（外部アプリ導線）
* Webアプリ内で回答作成・一言送信・逆質問送信をユーザーに促す場合に使用
* タップで LINE/X アプリを起動し、該当チャットスレッドへ遷移するよう設計
* リンク仕様：Deep Link（LINE Bot / X 連携）で実装
