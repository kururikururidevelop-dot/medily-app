# 画面ガイドライン / 共通仕様

# デザインコンセプト
## テーマ
* **医療系をイメージできる安心感のある緑系カラー**
* 清潔感・信頼感・易しさを重視
* 初めての人でも迷わない UI

## デザインキーワード
* **優しい（Gentle）**
* **直感的（Intuitive）**
* **クリーン（Clean）**
* **医療・信頼感（Healthcare & Trust）**

# タイポグラフィ
## ベースフォント
* **Noto Sans JP**（日本語）
* 可読性・医療文書に適した安定した書体

## 3.2 文字サイズ

| 用途    | px    | 説明               |
| ------- | ----- | ------------------ |
| H1      | 28–32 | 画面タイトル       |
| H2      | 22–24 | セクションタイトル |
| Body L  | 18    | メインテキスト     |
| Body M  | 16    | 標準               |
| Body S  | 14    | 説明文／タグ       |
| Caption | 12    | 補足情報           |


# スペーシング（余白）
医療系サービスは「詰めすぎない」ことで安心感を作る。

* **8px グリッドシステム**
* ベース：8px、16px、24px、32px
* セクション間は**32px**
* コンポーネント間は**16px**
* 行間は1.4〜1.6

# レスポンシブ設計
## ブレークポイント

| デバイス | width      | レイアウト |
| -------- | ---------- | ---------- |
| Mobile   | ～640px    | 1カラム    |
| Tablet   | 641–1024px | 1〜2カラム |
| PC       | 1025px〜   | 2〜3カラム |

## レスポンシブルール
* スマホは「親指操作」優先（主要操作は画面下部または中央）
* PCは横幅を使いすぎず、最大幅 **1024px** を基本とする
* カードは PC 2列、Tablet 2列、スマホ 1列

# コンポーネント共通仕様
## ボタン

| 状態     | 背景      | 文字      | 備考     |
| -------- | --------- | --------- | -------- |
| Primary  | `#2DB596` | White     | 角丸 8px |
| Hover    | `#4CCAB0` | White     |          |
| Disabled | `#DADADA` | `#8A8A8A` |          |

**タップ領域：最小44px**（Apple/Google推奨）

## 入力フォーム（Input）
* 枠線：`#DADADA`
* 角丸：8px
* フォーカス：`#2DB596` で 2px アウトライン
* プレースホルダー：Gray 500
* エラー：枠線を `#F75E63` に変更、下にエラーメッセージ表示

## カード
* 背景：White
* 影：`0 2px 8px rgba(0,0,0,0.05)`
* 角丸：12px（優しい印象）

## タブ
* アクティブ：Primary Dark 下線 2px
* 非アクティブ：Gray 300 下線
* アニメーション：100–150ms

# アイコンルール
* Material Icons または Feather Icons
* 線幅：2px
* アクセシビリティのため
  * 重要アイコンは色変化
  * 補助アイコンは Gray 500

# アクセシビリティ（重要）
* コントラスト比：**4.5:1 以上**
* ボタンサイズ：最低 44px
* 文字サイズ：最低 14px（説明文）
* 色だけで意味を伝えない（アイコン＋ラベル併記）
* フォーカスリングを必ず表示

# 主要画面に共通するレイアウト規則
## ヘッダー
* 高さ：56px
* 左にロゴ（Green系）
* 右にマイページアイコン
* スクロール時は shadow: `0 1px 4px rgba(0,0,0,0.06)`

## フッターメニュー（モバイル）
* 高さ：64px
* アイコン + ラベル
* 選択中は Primary カラー
